# アクセシビリティレビュー記録

生成日時: 2026-05-02
確認方法: ソース確認と既存E2Eの確認結果
確認種別: フロントエンドのアクセシビリティ実装確認
結果: 通過（制約あり）

この記録では、公開UI実装に対する小さなアクセシビリティ観点の確認結果を残す。
完全なWCAG監査とは主張しない。

確認用キー、provider response、cookie、secret値は含めない。

## 実装確認項目

- 検索入力のvalidation errorは `role="alert"` で通知する。
- 検索中、空結果、回答しない状態ではstatusまたはalert roleを使う。
- 回答パネルはstream中に `aria-busy` を設定する。
- UIには状態変化を伝えるpolite live regionを置く。
- RAG処理フローの詳細は、active stepの変化時に `aria-live="polite"` で通知する。
- citation操作は非interactive textではなくbuttonで行う。
- 引用番号を押すと、対応する根拠カードへフォーカスを移す。
- 根拠カードへのフォーカス移動はE2E testで確認する。
  - `tests/e2e/top-page.spec.ts`
- screenshot evidenceにはcitation focus状態を含める。
  - `docs/evidence/screenshots/ui-citation-focus-2026-04-30.png`
