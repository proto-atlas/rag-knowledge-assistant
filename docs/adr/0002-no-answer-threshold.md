# ADR 0002: No-Answer thresholdの扱い

状態: 採用
日付: 2026-05-02

## 日本語要約

この確認画面では、根拠が弱い質問に対して無理に回答を作らないことを優先します。実プロバイダー設定の `MIN_PROVIDER_VECTOR_SCORE = 0.55` は、この固定データ検証環境の25件fixtureに合わせた初期値であり、任意コーパスやVectorize一般に通用する本番しきい値ではありません。

## 背景

このデモでは、架空文書corpusに根拠がない回答を、根拠があるように表示しないことを優先します。固定データ経路と実プロバイダー経路ではscoreの仕組みが違うため、thresholdを同じ意味として扱いません。

## 決定

回答生成の前に検索側の回答しない判定を置き、実プロバイダー経路のしきい値調整は明示的に扱います。

現在の実プロバイダー既定値:

- `MIN_PROVIDER_VECTOR_SCORE = 0.55`
- optional override: `RAG_MIN_PROVIDER_VECTOR_SCORE`

この初期値は、架空の25件fixtureに対する実プロバイダー検索評価に合わせたものです。任意コーパス向けに調整済みの本番しきい値ではありません。より広い品質を説明する場合は、実プロバイダー経路のhit@kと回答しない判定の正解率を再測定します。

## 影響

利点:

- 根拠が弱い検索結果を、回答生成前に止められる。
- 回答しない判定の挙動を、回答文体とは分けて確認できる。
- thresholdをコード変更なしで調整できる。

トレードオフ:

- スコアしきい値だけで回答の正しさは証明できない。
- 初期値を調整済みとして扱う前に、実プロバイダー検索評価が必要になる。
